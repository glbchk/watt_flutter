# WATT
A Flutter application designed to simplify charging electric vehicles and offer charging stations for other users. The app provides a user experience from account creation to vehicle connection, allowing users to register, manage their EVs, and interact with charging infrastructure through an intuitive mobile interface.

# Features
- Secure authentication system
  - Sign in
  - Sign up
  - Password recovery
  - Guest access
- Guided onboarding experience
  - User profile setup
  - Vehicle registration
  - Charging station configuration
  - Payment method setup
- Vehicle management
  - Add and edit vehicles
  - Support for multiple vehicle brands
- Charging station management
  - Add and edit charging stations
  - Support of multiple charging station brands
  - Configuration of dates and times for booking charging stations
- Payment Method management
  - Add and edit credit cards
- Location services
  - Google Maps integration
  - Charging station discovery
  - Location-based functionality
- Charging Station booking
  - Time selection to book a charging station
  - Charging stage of booking
- Profile management
  - Edit profile
    - Management of:
      - Bookings
      - Reservations
      - Cars
      - Charging Stations
      - Payment Methods

# Tech Stack
- Flutter
- Dart
- Bloc/Cubit State Management
- Clean Architecture
- Firebase Authentication
- Cloud Firestore
- Google Maps API

# Architecture
This project represents my first attempt at building a production-style Flutter application. I aimed to follow Clean Architecture principles, organize features in a scalable manner, and implement state management using Bloc/Cubit to create a maintainable and testable codebase.

# Purpose
This project was created to explore and implement a complete EV-focused user journey, including authentication, onboarding, vehicle integration, and map-based services while following modern Flutter development best practices.
